Installation

Job Tracker runs locally as a web server at http://localhost:5000. Pick the method that best fits your environment.

Requirements

  • Linux / macOS: Python 3.10+, pip, git, bash.
  • Windows: nothing — the NSIS installer bundles an embedded Python. For manual install, Python 3.10+.
  • Optional: pystray + Pillow for the system tray icon.

Linux / macOS (script)

The recommended approach is the install.sh script:

git clone https://github.com/rafaelkrause/job_tracker.git
cd job_tracker
./install.sh

The script:

  1. Verifies Python 3.10+ is available.
  2. Creates a virtual environment at .venv/.
  3. Installs dependencies from requirements.txt.
  4. Optionally installs pystray + Pillow for the tray icon.
  5. Creates a .desktop shortcut (if a graphical environment is detected).

Then launch the app with:

./job-tracker.sh
# or
python3 run.py

Manual install (Linux / macOS)

If you prefer to control each step:

git clone https://github.com/rafaelkrause/job_tracker.git
cd job_tracker

python3 -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt

# optional: tray support
pip install pystray Pillow

python3 run.py

Use python3 run.py --no-browser to start without opening the browser automatically.

Windows — NSIS installer

Download JobTracker-Setup-X.Y.Z.exe from the releases page and run it.

  • Per-user install: no UAC required. Default path %LOCALAPPDATA%\Programs\JobTracker.
  • Embedded Python: a portable Python is bundled — nothing installs globally.
  • Data: configuration and history live in %APPDATA%\JobTracker.
  • Shortcuts: desktop and Start Menu are optional wizard steps.
  • Windows Service: optional component using NSSM, starts the tracker at boot (UAC only if enabled).

Windows — manual install

  1. Install Python 3.10+. Check Add Python to PATH.
  2. Download the code (release ZIP or git clone).
  3. In PowerShell: 
    cd job_tracker
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python run.py

Build the installer from source

The Windows installer is generated on a Linux host with NSIS:

sudo apt install nsis
./installer/build_installer.sh 1.0.0
# output: installer/JobTracker-Setup-1.0.0.exe

The workflow .github/workflows/build-installer.yml does this automatically when a v* tag is pushed.

Run at startup

Linux — systemd (user unit)

Create ~/.config/systemd/user/job-tracker.service:

[Unit]
Description=Job Tracker
After=network.target

[Service]
Type=simple
WorkingDirectory=%h/job_tracker
ExecStart=%h/job_tracker/.venv/bin/python run.py --no-browser
Restart=on-failure

[Install]
WantedBy=default.target
systemctl --user daemon-reload
systemctl --user enable --now job-tracker.service

Windows — service via NSSM

The official installer offers a service component. Alternatively, after a manual install, use NSSM:

nssm install JobTracker "C:\path\job_tracker\.venv\Scripts\python.exe" run.py --no-browser
nssm start JobTracker

Update

Linux / macOS

cd job_tracker
git pull
source .venv/bin/activate
pip install -r requirements.txt --upgrade

Windows

Download and run the new JobTracker-Setup-X.Y.Z.exe. The installer preserves your configuration and data in %APPDATA%\JobTracker.

Uninstall

  • Linux/macOS: rm -rf job_tracker/ and, if applicable, systemctl --user disable --now job-tracker.service.
  • Windows: Control Panel → Programs → Job Tracker → Uninstall, or via Settings → Apps.
  • To also remove data: delete the data/ folder (Linux) or %APPDATA%\JobTracker (Windows).

Troubleshooting

Port 5000 in use

Change the port in config.json:

{ "port": 5050 }

Or via the /settings page.

Tray icon does not appear

Install the optional dependencies:

pip install pystray Pillow

On Linux, some desktops (pure GNOME) need the AppIndicator extension.

“ModuleNotFoundError: flask”

The venv is not active. Run source .venv/bin/activate (Linux/macOS) or .venv\Scripts\Activate.ps1 (Windows).

Windows blocks the .exe (SmartScreen)

Click More info → Run anyway. Unsigned installers trigger this warning.
