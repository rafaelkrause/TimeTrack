User guide

Job Tracker is built for people who need to log hours throughout the day without interrupting the flow. This guide covers every feature of the app.

The UI is in Brazilian Portuguese (pt-BR) by design. This guide translates the concepts and navigation for English-speaking readers.

First run

  1. Start the server (python3 run.py or ./job-tracker.sh).
  2. The browser opens at http://localhost:5000.
  3. On the first run, a config.json is generated with sensible defaults (shifts 09–12 / 13–18 Mon–Fri, auto theme, port 5000, target 90%).
  4. Adjust shifts and preferences in Configurações / Settings ( icon).

Core concepts

ConceptDescription
ActivityA block of work with description, start time, pauses and end time.
PauseInterval inside an activity. Time spent paused is subtracted from total duration.
ShiftExpected working hours per weekday. Can contain multiple blocks (e.g. morning/afternoon).
Target %Percentage of the shift you plan to actually log as productive activity.
StateState machine: active → paused → active → completed.

Working with activities

Start an activity

Type a description in the main field and click Iniciar (Start). If another activity is already running, it gets finalized automatically — no blocking prompt.

Pause and resume

Use Pausar (Pause) and Retomar (Resume). The counter stops/resumes; total duration is always elapsed time minus paused time.

Stop

Click Parar (Stop). The activity becomes completed and joins the day's history.

Switch tasks quickly

Start a new activity directly from the description input. The previous one is finalized at the same instant.

Daily dashboard

The home page shows:

  • Day timeline — hourly view with each activity placed at its real start/end. Range is derived from the day's shift config + 30min padding.
  • Current activity — description, client-side counter incremented in the browser between polls (30s).
  • Shift progress bar — only counts hours already elapsed in the current day, so future hours do not drag the bar down.
  • Target — how much of your target (% of the shift) has been logged as productive activity.
  • Day summary — total logged and chronological list of completed activities.

Navigate to other days via the date picker at the top.

Shifts

Each weekday can have zero, one or more time blocks. Default example:

{
  "monday":    [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "tuesday":   [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "wednesday": [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "thursday":  [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "friday":    [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "saturday":  [],
  "sunday":    []
}
  • Edit via UI under Configurações → Turnos or directly in config.json.
  • Time format: HH:MM (24h). Validated on the server.
  • Days with no blocks appear as free, with no hour target.

Configuration

The config.json file at the project root holds all preferences:

KeyTypeDescription
shiftsobjectShifts per weekday (see above).
themestring"light", "dark" or "auto".
portnumberHTTP port for the local server. Default 5000.
target_percentagenumberTarget % of the shift to be logged. Default 90.

Unknown keys are ignored. Writes to disk are atomic: temp file → fsync → os.replace().

System tray (optional)

With pystray and Pillow installed, Job Tracker shows a tray icon with quick actions:

  • Pause / Resume current activity
  • Stop activity
  • Open in browser
  • Quit

Tray actions talk to Flask over local HTTP — the same API the UI uses.

Export to iClips

The Exportar (Export) page (or the /api/export endpoint) generates CSV/TSV with completed activities over a date range.

  • The format is optimized for manual paste into iClips — there is no official iClips API.
  • Running or paused activities are not exported.
  • TSV makes it easy to paste into spreadsheets / table grids.

Where data lives

  • Linux/macOS: data/YYYY-MM.json in the project folder.
  • Windows (installer): %APPDATA%\JobTracker\data\YYYY-MM.json.
  • One file per month. Files older than 12 months are pruned automatically on start.
  • On corrupted JSON, the app renames the file to .corrupted and continues with empty state for that month — you can recover manually.

REST API

Handy for scripts, automations or custom integrations.

POST /api/activity/start    body: {"description": "..."}
POST /api/activity/pause
POST /api/activity/resume
POST /api/activity/stop
GET  /api/activity/current
GET  /api/dashboard?date=YYYY-MM-DD
GET  /api/export?from=YYYY-MM-DD&to=YYYY-MM-DD&format=csv|tsv
GET  /api/shifts
PUT  /api/shifts
PUT  /api/config

Request body is limited to 64 KB. description is at most 500 characters.

Quick example (curl):

curl -X POST http://localhost:5000/api/activity/start \
     -H "Content-Type: application/json" \
     -d '{"description": "Client XYZ meeting"}'

curl http://localhost:5000/api/activity/current

Usage tips

  • Keep a browser tab open on the dashboard throughout the day.
  • Use the tray to pause quickly without leaving what you are doing.
  • Short, consistent descriptions make it easier to log into iClips on Friday.
  • Set up a system keyboard shortcut to focus the Job Tracker tab.
  • If you run it as a service, remember the --no-browser flag.
