Guia de uso

O Job Tracker é pensado para quem precisa apontar horas durante o dia sem interromper o fluxo. Este guia cobre todas as funcionalidades do app.

Primeiro acesso

  1. Rode o servidor (python3 run.py ou ./job-tracker.sh).
  2. O navegador abre em http://localhost:5000.
  3. Na primeira execução, um config.json é gerado com valores padrão (turnos 09–12 / 13–18 de seg a sex, tema automático, porta 5000, meta 90%).
  4. Ajuste turnos e preferências em Configurações (ícone ).

Conceitos centrais

ConceitoDescrição
AtividadeUm bloco de trabalho com descrição, hora de início, pausas e hora de fim.
PausaIntervalo dentro de uma atividade. O tempo em pausa é subtraído da duração total.
TurnoFaixa de horário esperada de trabalho por dia da semana. Pode ter múltiplos blocos (manhã/tarde).
Meta (target %)Porcentagem do turno que você deseja efetivamente apontar como atividade produtiva.
EstadoMáquina de estados: active → paused → active → completed.

Trabalhando com atividades

Iniciar uma atividade

Digite uma descrição no campo principal e clique em Iniciar. Se já houver uma atividade em curso, ela é finalizada automaticamente — sem prompt bloqueante.

Pausar e retomar

Use os botões Pausar e Retomar. O contador para/segue o relógio; a duração total é sempre o tempo decorrido menos o tempo em pausa.

Finalizar

Clique em Parar. A atividade é marcada como completed e entra no histórico do dia.

Trocar rapidamente de tarefa

Comece uma nova atividade diretamente pelo campo de descrição. A atividade anterior é finalizada automaticamente no mesmo instante.

Dashboard diário

A página inicial mostra:

  • Timeline do dia — visualização horária com cada atividade posicionada no seu horário real de início/fim. Faixa derivada do turno configurado + 30min de padding.
  • Atividade atual — descrição, cronômetro local incrementado no navegador entre polls (30s).
  • Barra de progresso do turno — só conta horas já decorridas no dia corrente, então horas futuras não puxam o progresso pra baixo.
  • Meta — quanto da sua meta (% do turno) já foi apontada como atividade produtiva.
  • Resumo do dia — total apontado e lista cronológica de atividades finalizadas.

Você pode navegar para outros dias via o seletor de data no topo.

Turnos

Cada dia da semana pode ter zero, um ou mais blocos de horário. Exemplo padrão:

{
  "monday":    [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "tuesday":   [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "wednesday": [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "thursday":  [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "friday":    [{"start": "09:00", "end": "12:00"}, {"start": "13:00", "end": "18:00"}],
  "saturday":  [],
  "sunday":    []
}
  • Edite pela UI em Configurações → Turnos ou direto no config.json.
  • Formato dos horários: HH:MM (24h). Validado no servidor.
  • Dias sem blocos aparecem como livres, sem meta de horas.

Configuração

O arquivo config.json na raiz do projeto guarda todas as preferências:

ChaveTipoDescrição
shiftsobjetoTurnos por dia da semana (ver acima).
themestring"light", "dark" ou "auto".
portnúmeroPorta HTTP do servidor local. Padrão 5000.
target_percentagenúmeroMeta de % do turno a ser apontada. Padrão 90.

Chaves desconhecidas são ignoradas. Escritas em disco são atômicas: arquivo temporário → fsync → os.replace().

Bandeja do sistema (opcional)

Com pystray e Pillow instalados, o Job Tracker mostra um ícone na bandeja com ações rápidas:

  • Pausar / Retomar atividade atual
  • Parar atividade
  • Abrir no navegador
  • Sair

As ações da bandeja conversam com o Flask via HTTP local — a mesma API usada pela UI.

Exportação para iClips

A página Exportar (ou o endpoint /api/export) gera CSV/TSV com atividades finalizadas de um intervalo de datas.

  • O formato é otimizado para colar manualmente no iClips — não há API oficial do iClips.
  • Atividades em andamento ou pausadas não são exportadas.
  • TSV facilita a colagem direta em planilhas / grids tabulares.

Onde os dados ficam

  • Linux/macOS: data/YYYY-MM.json na pasta do projeto.
  • Windows (instalador): %APPDATA%\JobTracker\data\YYYY-MM.json.
  • Um arquivo por mês. Arquivos com mais de 12 meses são podados automaticamente no start.
  • Em caso de JSON corrompido, o app renomeia o arquivo para .corrupted e segue com estado vazio daquele mês — você pode recuperar manualmente.

API REST

Útil para scripts, automações ou integrações caseiras.

POST /api/activity/start    body: {"description": "..."}
POST /api/activity/pause
POST /api/activity/resume
POST /api/activity/stop
GET  /api/activity/current
GET  /api/dashboard?date=YYYY-MM-DD
GET  /api/export?from=YYYY-MM-DD&to=YYYY-MM-DD&format=csv|tsv
GET  /api/shifts
PUT  /api/shifts
PUT  /api/config

O corpo é limitado a 64 KB. description tem no máximo 500 caracteres.

Exemplo rápido (curl):

curl -X POST http://localhost:5000/api/activity/start \
     -H "Content-Type: application/json" \
     -d '{"description": "Reunião com cliente XYZ"}'

curl http://localhost:5000/api/activity/current

Dicas de uso

  • Deixe uma aba do navegador aberta no dashboard ao longo do dia.
  • Use o tray para pausar rapidamente sem sair do que está fazendo.
  • Descrições curtas e consistentes facilitam o apontamento no iClips na sexta.
  • Configure um atalho de teclado do sistema para focar a aba do Job Tracker.
  • Se for rodar como serviço, lembre-se do --no-browser.
